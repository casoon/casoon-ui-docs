# Getting Started

## Einführung

Die Casoon UI Library ist eine moderne, flexible und zugängliche Komponentenbibliothek für Webanwendungen. Sie bietet:

- **Modulare Komponenten**: Jede Komponente kann unabhängig verwendet werden
- **Responsive Design**: Optimiert für alle Bildschirmgrößen
- **Zugänglichkeit**: WCAG 2.1 konform
- **Theming**: Anpassbare Designs und Farben
- **Performance**: Optimiert für schnelle Ladezeiten

## Installation

### Voraussetzungen

- Node.js (Version 14 oder höher)
- npm, yarn oder pnpm
- Git

### Schritt 1: Repository klonen

```bash
# Neues Repository initialisieren
git init

# Repository klonen
git clone https://github.com/your-username/casoon-ui-lib.git

# In das Projektverzeichnis wechseln
cd casoon-ui-lib
```

### Schritt 2: Abhängigkeiten installieren

```bash
# Mit npm
npm install casoon-ui-lib

# Oder mit yarn
yarn add casoon-ui-lib

# Oder mit pnpm
pnpm add casoon-ui-lib
```

### Schritt 3: In Ihrem Projekt verwenden

```css
/* Basis-Styles importieren */
@import 'casoon-ui-lib/styles/base.css';

/* Benötigte Module importieren */
@import 'casoon-ui-lib/modules/button.module.css';
@import 'casoon-ui-lib/modules/input.module.css';
@import 'casoon-ui-lib/modules/modal.module.css';

/* Theme importieren */
@import 'casoon-ui-lib/themes/default.css';
```

## Framework Integration

### React

```jsx
import 'casoon-ui-lib/styles/base.css';
import 'casoon-ui-lib/modules/button.module.css';

function MyComponent() {
  return (
    <button className="c-button c-button--primary">
      Klick mich
    </button>
  );
}
```

### Vue

```vue
<template>
  <button class="c-button c-button--primary">
    Klick mich
  </button>
</template>

<script>
import 'casoon-ui-lib/styles/base.css';
import 'casoon-ui-lib/modules/button.module.css';

export default {
  name: 'MyComponent'
}
</script>
```

### Angular

```typescript
import { Component } from '@angular/core';
import 'casoon-ui-lib/styles/base.css';
import 'casoon-ui-lib/modules/button.module.css';

@Component({
  selector: 'app-my-component',
  template: `
    <button class="c-button c-button--primary">
      Klick mich
    </button>
  `
})
export class MyComponent {}
```

## Best Practices

### Komponenten-Verwendung

1. **Modularität**
   - Importieren Sie nur die benötigten Komponenten
   - Vermeiden Sie globale Styles
   - Nutzen Sie CSS-Module für bessere Isolation

2. **Zugänglichkeit**
   - Verwenden Sie semantische HTML-Elemente
   - Implementieren Sie ARIA-Attribute
   - Stellen Sie ausreichenden Kontrast sicher

3. **Responsive Design**
   - Testen Sie auf verschiedenen Bildschirmgrößen
   - Nutzen Sie die eingebauten Breakpoints
   - Implementieren Sie mobile-first Ansätze

### Performance-Optimierung

1. **CSS-Optimierung**
   ```css
   /* Nur benötigte Module importieren */
   @import 'casoon-ui-lib/modules/button.module.css';
   @import 'casoon-ui-lib/modules/input.module.css';
   ```

2. **Tree Shaking**
   - Nutzen Sie moderne Bundler wie Webpack oder Rollup
   - Aktivieren Sie CSS-Module für bessere Optimierung
   - Entfernen Sie ungenutzte Styles

3. **Lazy Loading**
   ```javascript
   // Dynamische Imports für große Komponenten
   import('casoon-ui-lib/modules/modal.module.css');
   ```

## Häufige Fragen (FAQ)

### Installation

**F: Wie aktualisiere ich die Bibliothek?**
A: Führen Sie `npm update casoon-ui-lib` aus oder aktualisieren Sie die Version in Ihrer `package.json`.

**F: Kann ich die Bibliothek ohne npm verwenden?**
A: Ja, Sie können die CSS-Dateien direkt von einem CDN einbinden.

### Komponenten

**F: Wie passe ich das Design an?**
A: Nutzen Sie CSS-Variablen oder überschreiben Sie die Styles mit höherer Spezifität.

**F: Sind die Komponenten mit allen Browsern kompatibel?**
A: Die Bibliothek unterstützt alle modernen Browser und IE11 mit Polyfills.

## Troubleshooting

### Häufige Probleme

1. **Styles werden nicht angewendet**
   - Überprüfen Sie die Import-Pfade
   - Stellen Sie sicher, dass CSS-Module aktiviert sind
   - Überprüfen Sie die Spezifität Ihrer Styles

2. **Komponenten funktionieren nicht**
   - Überprüfen Sie die Browser-Kompatibilität
   - Stellen Sie sicher, dass alle Abhängigkeiten installiert sind
   - Überprüfen Sie die Konsole auf Fehler

3. **Performance-Probleme**
   - Reduzieren Sie die Anzahl der importierten Module
   - Nutzen Sie Code-Splitting
   - Optimieren Sie Ihre Bundler-Konfiguration

### Debugging
